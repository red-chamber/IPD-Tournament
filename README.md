# 背景

1980 年代，政治学家罗伯特·阿克塞尔罗德（Robert Axelrod）组织了两场著名的计算机锦标赛：来自全球的博弈论专家提交策略程序，在**重复囚徒困境**中互相博弈。

实验结果出人意料：最简单的策略 **Tit for Tat（以牙还牙）** 在两届比赛中均获得冠军。其成功的四个特征是：

1. **善良** — 从不率先背叛
2. **报复性** — 一旦被背叛立刻还击
3. **宽容** — 对方恢复合作后也恢复合作
4. **清晰** — 行为模式容易被识别

道金斯在《自私的基因》中引用这个实验，论证了一个核心观点：在重复互动的框架下，**合作行为可以从自私基因中自然涌现**。

# 功能概览

- **循环锦标赛** — 所有选定策略两两对战，实时更新排行榜
- **种群生态演化** — 策略按得分比例繁殖/淘汰，模拟自然选择
- **实时交互** — 基于 SocketIO 推送比赛进展，图表和表格动态更新
- **自定义策略** — 在浏览器中直接编写 Python 策略代码，无需重启服务
- **可调参数** — 轮数、噪音水平、生态代数自由调节
- **可视化分析** — 柱状图/线图 + 详情面板 + 比赛日志

# 两种模式

## 循环锦标赛

所有选中的策略两两对战一轮（round-robin），每场比赛进行 N 轮重复囚徒困境。

**流程：**
1. 选择参与的策略
2. 设定轮数和噪音
3. 点击"开始锦标赛"
4. 每场比赛结束后即时更新排行榜

**输出：**
- 实时柱状图（总得分排名）
- 排行榜表格（总分、均分、胜率、合作率）
- 比赛记录日志
- 详情面板（合作/背叛比例 + 前 50 回合出招序列）

**参考收益矩阵：**

| A \ B | B合作     | B背叛     |
| ----- | ------- | ------- |
| A合作   | A+3，B+3 | B+5     |
| A背叛   | A+5     | A+1，B+1 |


## 种群生态

模拟达尔文式的自然选择：每个策略的初始种群数量相同，每代中所有个体随机配对比赛，按平均得分（适合度）比例繁殖。

**流程：**
1. 选择物种
2. 设定轮数、噪音、代数
3. 点击"开始生态模拟"
4. 观察种群数量随代数的演化

**输出：**
- 实时线图（各物种种群数量变化曲线）
- 种群数量排行榜

**繁殖规则：**

> 每代的计算流程：

1. 随机配对

所有存活的个体全部随机两两配对，每对打一场比赛。

2. 计算平均得分（适合度）

```text
策略 A 的适合度 = 该代中所有 A 个体的总得分 ÷ A 个体的数量
```

3. 所有策略的适合度取平均

```text
平均适合度 = 所有正数适合度的平均值
```

4. 下一代种群数量

```text
策略 A 的下一代数量 = max(5, A 的当前种群 × (A 的适合度 ÷ 平均适合度))
```

| 参数   | 值                                       | 说明              |
| ---- | --------------------------------------- | --------------- |
| 初始种群 | **100** 每个策略                            | 所有物种起点相同        |
| 种群下限 | **5**                                   | 低于此数不再缩小，但仍可能灭绝 |
| 灭绝条件 | 适合度为 0（全部个体得分 0）                        | 种群直接降为 0        |

# 内置策略

| ID              | 名称               | 行为                      |
| --------------- | ---------------- | ----------------------- |
| AlwaysCooperate | Always Cooperate | 永远选择合作                  |
| AlwaysDefect    | Always Defect    | 永远选择背叛                  |
| TitForTat       | Tit for Tat      | 第一轮合作，之后复制对手上一轮         |
| SuspiciousTFT   | Suspicious TFT   | 第一轮背叛，之后复制对手上一轮         |
| TitForTwoTats   | Tit for Two Tats | 更宽容，只在对方连续背叛两次后才报复      |
| Friedman        | Friedman         | 合作直到被背叛一次，之后永远背叛（永不原谅）  |
| Joss            | Joss             | 类似 TFT，但有 10% 概率随机背叛    |
| GenerousTFT     | Generous TFT     | 类似 TFT，但有 10% 概率原谅对方的背叛 |
| Random          | Random           | 以 50% 概率随机选择合作/背叛       |

# 自定义策略

在浏览器左侧的代码编辑器中编写 Python 函数。函数签名固定：

```python
def move(my_history, opp_history):
    """自定义策略函数
    
    参数:
        my_history: list[Move]    — 自己此前的出招序列
        opp_history: list[Move]   — 对手此前的出招序列
        Move 枚举: Move.COOPERATE, Move.DEFECT
    
    返回:
        Move.COOPERATE 或 Move.DEFECT
    
    可用模块: random
    """
    if not opp_history:
        return Move.COOPERATE  # 首轮友好
    
    # 示例：检测对手背叛率
    defects = sum(1 for m in opp_history if m == Move.DEFECT)
    if defects > len(opp_history) * 0.3:
        return Move.DEFECT
    
    return opp_history[-1]  # 否则模仿对手上一步
```

**提示：**

- 可使用 `random` 模块实现概率性策略
- 策略名称可以自定义
- 多个自定义策略可以同时参与
- 策略代码在服务端 `exec()` 执行

| 参数 | 范围 | 默认值 | 说明 |
|---|---|---|---|
| 轮数 | 20–500 | 200 | 每场比赛的重复博弈回合数 |
| 噪音 | 0%–30% | 0% | 每回合以该概率随机翻转出招（模拟误操作） |
| 代数 | 5–100 | 30 | 仅生态模式，模拟的总世代数 |

**噪音的作用：**

噪音是理解 TFT 弱点的关键。在无噪音环境下 TFT 表现出色，但一旦引入误操作，TFT 会陷入无休止的互相报复循环。Generous TFT（宽容版）在噪音环境下表现更好——这也解释了真实世界中"原谅"的进化优势。

# 常见问题

### Q: 为什么有些策略在生态模式中显示为 0？

生态模式下表格显示的是**种群数量**（不是得分）。当某策略的种群降至 0，表示该物种已灭绝，这是模拟中的正常生态现象。

### Q: 修改代码后需要重启吗？

需要。修改 `app.py` 或 `index.html` 后，需停止并重新启动 Flask 服务。

### Q: 如何添加更多自定义策略？

在左侧编辑器中编写代码，点击"添加"后即可在下一轮锦标赛中使用。也可以直接在 `app.py` 的 `CLASSIC_STRATEGIES` 字典中添加新的策略类。
